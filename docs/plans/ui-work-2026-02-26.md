## Commit plan

1. **Name screen parts**
	- Update layout names in `src/cli/tui.ts` comments.
	- Update `docs/tui.md` terminology.

2. **Title bar visibility + session context**
	- Give title bar a visible background treatment.
	- Ensure title row includes session context (session name fallback when title missing).

3. **Model visibility (short human label)**
	- Surface model name clearly in activity area.
	- Convert full model IDs to short labels (e.g. `Opus 4.6`, `Codex 5.3`).

4. **Tab bar redesign**
	- Render tabs as `[1 .hal] 2 lines 3 lippu.1 ...` shape.
	- Show directory-only labels.
	- Disambiguate duplicate dirs with `.1`, `.2`, ...
	- Active tab bright, inactive tabs dim/readable.
	- Remove status-line dash ruler background.

5. **Tab activity indicator**
	- Add activity marker between tab number and dir label for non-active tabs with unseen output.

6. **Prompt echo styling**
	- Prompt echoes should be grey text on every line, not just bars around block.

7. **Tool line color continuity**
	- Fix multi-line tool output so color is stable across all lines, not just first lines.

8. **Prompt block color continuity**
	- Fix multi-line prompt echo so style applies to all wrapped/stored lines.

9. **"Bright initial words" artifact**
	- Fix chunk styling boundary behavior causing intra-line brightness changes.
	- Keep output style stable regardless of chunk boundaries.

## Structural note

While implementing, move TUI chrome formatting helpers out of `tui.ts` into `src/cli/tui/format/*` to keep responsibilities separate and prevent `tui.ts` bloat.
